通用库

库包含的代码通常被多个程序所需。这样做的好处是每个程序无需重复编写代码（从而避免引入缺陷），只需调用系统上已安装库中的函数即可。最明显的库集合示例是 Glibc，它是在 LFS 手册中安装的，包含了程序使用的所有 C 库函数。

库有两种类型：静态库和共享库。共享库（通常为 libXXX.so）在运行时从共享副本加载到内存中（因此得名）。静态库（libXXX.a）则实际上被链接到程序可执行文件本身，从而使程序文件更大。通常，您会在系统上同时找到同一库的静态和共享副本。

通常，仅在安装需要库所提供功能的软件时，才需要安装这些库。在 BLFS 手册中，每个软件包都附有（已知）依赖项列表。因此，您可以在安装该程序之前确定需要哪些库。如果您未按照 BLFS 说明安装某些软件，通常 README 或 INSTALL 文件中会包含程序需求的详细信息。

有些库几乎 所有人 在某个时候都会需要。本章列出了这些库以及其中一些其他库，并解释了您可能希望安装它们的原因。
